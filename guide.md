## 웹개발 과정 (프론트엔드 ~ 백엔드)

# 프로젝트를 위해 필요한 선택지

```
1. 서버선택
2. 백엔드 언어 선택
3. 데이터베이스 선택
4. 아키텍트 구상
5. 프론트엔드 선택 (SPA, 링크방식)
```

# 기술사항

```
1.OS선택 (window server_유료, linux server_**취업을 위해 추천!)
    -window
    -centos(**취업을 위해 추천!)
    -ubuntu(무료: 개인 작업을 위해 추천!) [[hanpixel이 사용]]
    
    
2. 웹서버(웹언어를 해석할 수 있게 도와주는 역할 + 서버의 역할)
    -apache (**취업을 위해 추천!)
    -nginx (떠오르는 샛별_ 스타트업에서 사용) [[hanpixel이 사용]]
    
    -node (javaScript로 되어있어서 프론트엔드가 백앤드를 하도록 도움)
    --> 서버를 제어할 수 있는 능력을 가진 언어
    --> socket io를 공부하자 (chatting을 만들 수 있다) [[hanpixel이 사용]]
    
    
3. 백엔드 웹개발 언어 선택 
    -php [[hanpixel이 사용]]
    -python(추천) [[hanpixel이 사용]]
    -java & jsp(**취업을 위해 추천)
    -node [[hanpixel이 사용]]
    
    -go lang 
    --> 구글에서 알파고를 만들 때 사용한 언어


4. 데이터베이스 선택
    -oracle DB
    
    -mysql & mariaDB (**취업을 위해 추천) [[hanpixel이 사용]]
    --> mysql이 oracle에 인수되면서, 유료화 되었다.
    --> mysql 개발자가 무료인 mariaDB를 새로 만들었다.
    --> 명령어(sql문법)가 둘이 똑같다.
    
    -mongoDB : NoSQL
    --> 명령어가 없어서 NoSQL
    --> 검색이 빠르다 : 빅데이터에서 사용
    
    
5. 개발방식
    -restful api (**취업을 위해 추천) [[hanpixel이 사용]]
    --> 멀티 디바이스가 가능하다.
    
    -MVC : model - controller - view)
    
    
6. 프론트엔드 
    -HTML, CSS, JS 는 필수!!
    
    
7. 프론트엔드 개발방식 선택
    -SPA page
    --> react(매우 어려움) [[hanpixel이 사용]]
    --> angular(mvc패턴을 알면 가능)
    --> vue(초보에게추천)
        
    -line 방식(페이지이동 방식)
    
    
8. UI/UX
    -(고정형 웹)
    -반응형
    -하이브리드형 (웹 + 모바일웹)
    -네이티브 (웹 + 앱)
```
